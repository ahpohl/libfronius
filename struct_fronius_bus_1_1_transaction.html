Detailed Description

A single Modbus register-read request submitted to the bus queue.

Created by device fetch methods and queued via FroniusBus::submit(). The bus thread fulfils promise after the read completes or times out.

Note
promise is move-only; Transaction is therefore move-only too.

int FroniusBus::Transaction::secTimeout {0}

Per-transaction response timeout (whole seconds component).

Copied from the device's ModbusDeviceConfig::secTimeout and applied via modbus_set_response_timeout() before each read, so different slaves on the same bus can run with different timeouts.
