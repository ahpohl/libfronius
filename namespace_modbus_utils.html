Retrieves the remote peer's IP address and port from a connected socket.

Extracts peer connection information from a socket file descriptor, supporting both IPv4 and IPv6 protocols. Protocol-independent and works for both client (master) and server (slave) socket roles.

Parameters
socketConnected socket file descriptor (e.g., returned by accept() or after a successful connect()).
Returns
A FroniusTypes::RemoteEndpoint with:
  • ip: Remote IP address as a string (dotted-decimal IPv4 or colon-separated IPv6 format).
  • port: Remote port number as an integer. Returns {"unknown", 0} if the socket is invalid or peer information cannot be retrieved.

Encode a floating-point value into an integer register plus a SunSpec scale-factor register.

Scales realValue by 10^decimals, rounds to the nearest integer, and writes the result into the register at reg.ADDR. The corresponding scale factor register at sf.ADDR is written as -decimals (a signed INT16), following the SunSpec convention where the reconstructed value equals register_value × 10^sf.

Supported register types for reg: INT16, UINT16, UINT32, UINT64.

Example — storing 230.5 V with 1 decimal place into a UINT16 voltage register and its SF register:

packToModbus(mapping, M20X::PHVPHA, M20X::V_SF, 230.5, 1);
// tab_registers[PHVPHA.ADDR] = 2305
// tab_registers[V_SF.ADDR] = 0xFFFF (-1 as INT16)
M20X::V_SF
constexpr Register V_SF(40084, 1, Register::Type::INT16)
Voltage scale factor.
M20X::PHVPHA
constexpr Register PHVPHA(40077, 1, Register::Type::INT16)
AC voltage phase A to neutral Unit: Volts [V].

template<typename T >
std::expected< void, ModbusError > ModbusUtils::packToModbus ( modbus_mapping_t *  dest,
Register  reg,
T  value 
)

Write a typed value into a Modbus register mapping.

Encodes value into the register(s) starting at reg.ADDR inside dest, using the encoding implied by reg.TYPE:

Register::Type Accepted T Encoding
INT16 integral cast to int16_t, stored as 1 register
UINT16 integral stored as 1 register
UINT32 integral big-endian word order, 2 registers
UINT64 integral big-endian word order, 4 registers
FLOAT floating-point IEEE 754 ABCD byte order, 2 registers
STRING std::string ASCII, high byte first per register, zero-padded to reg.NB registers

The template is intentionally permissive — mismatched combinations (e.g. passing a float for an INT16 register) compile but are no-ops at runtime. Prefer using the correct type for the register's declared type.

Template Parameters
TType of the value to encode. Must be integral, floating-point, or std::string, matching the register's declared type.
Parameters
destTarget Modbus mapping. Must not be null.
regRegister definition specifying the address, width, and type.
valueValue to encode.
Returns
An empty expected on success, or an unexpected ModbusError if:
  • dest or its tab_registers pointer is null (EINVAL)
  • reg.TYPE is UNKNOWN or unsupported (EINVAL)
  • A STRING value exceeds the register's declared capacity (EINVAL)
